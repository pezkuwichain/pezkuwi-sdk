Pezkuwi <> Kusama Bridge

Our bridge connects two parachains - Kusama Bridge Hub and Pezkuwi Bridge Hub. Messages that are sent over bridge have XCM format and we are using existing architecture to dispatch them. Since both Pezkuwi, Kusama and their parachains already have means to exchange XCM messages within the same consensus system (HRMP, VMP, ...), it means that we are able to connect all those chains with our bridge.

In our architecture, the lane that is used to relay messages over the bridge is determined by the XCM source and destinations. So e.g. bridge between Asset Hubs Pezkuwi and Kusama (and opposite direction) will use the lane 00000000, bridge between some other Pezkuwi Parachain and some other Kusama Parachain will use the lane 00000001 and so on.
